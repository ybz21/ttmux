# 文件树右键菜单需求草案

状态：WIP，先声明计划避免重复开发。

## 背景

当前前端文件树的右键能力不完整，实际使用时只能覆盖很窄的删除目录场景。文件树需要补齐接近常规文件管理器的上下文操作，减少用户在终端中手动执行文件操作的频率。

## 功能范围

- 在文件和目录节点上提供右键菜单。
- 支持重命名文件或目录。
- 支持将文件或目录复制到用户指定路径。
- 支持在当前目录上传文件。
- 支持下载文件；下载目录时由后端打包为 Zip。
- 支持查看属性，包括类型、路径、大小、修改时间等元数据。
- 保留现有删除能力，并让文件和目录的操作入口一致。

## 实现约束

- 所有新增用户可见文案必须通过 `frontend/src/i18n/`，同时补齐 `zh-CN` 与 `en-US`。
- 前端交互遵循 `docs/design/web/04-frontend.md`，右键菜单优先使用 Ant Design `Dropdown`。
- 文件写操作应复用或扩展现有文件 API，后端返回稳定错误信息，前端负责本地化展示。
- 下载目录需要使用 Zip 响应，并给出合理的下载文件名。

## 验收计划

- 前端文件树中对文件和目录右键均能打开菜单。
- 重命名、复制到路径、上传到当前目录、下载、属性查看和删除均有明确成功/失败反馈。
- 目录下载得到可解压的 Zip 包。
- 运行 `cd frontend && npm run i18n:check`。
- 开发完成后运行 `scripts/dev/quality/check.sh quick`；若涉及运行时行为进入 PR 更新，运行 `scripts/dev/quality/check.sh full`。

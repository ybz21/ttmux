# 01 · 总体设计

← 返回 [README](./README.md)

## 1. 背景

ttmux 当前是纯 Bash CLI，状态全部落在文件 + tmux server，且已支持 `--json` 输出。
因此 Web 化的核心逻辑可**直接复用 CLI**：读 = 调 `ttmux <cmd> --json` 解析，写 = 调对应子命令。
不重写任何编排引擎，保证 Web 与 CLI 行为永远一致。

## 2. 目标

### 2.1 必须（MUST）
- **三端统一**：同一套代码、同一份 URL，在手机/平板/电脑浏览器上都能良好操作，布局自适应。
- **远程访问**：外网手机可访问，无需 SSH。
- **看**：实时查看 session、group 进度、agent 状态、日志、终端。
- **控**：spawn 任务 / agent、send 指令、kill、env set/rm/push。
- **进终端**：浏览器内 attach 到 session 实时终端（看 Claude agent、批准操作）。
- **认证**：强制登录，绝不裸暴露公网。

### 2.2 非目标（暂不做）
- 多用户 / 权限分级（先单用户）。
- 持久化数据库（先用现有文件 + 内存缓存）。
- 任务依赖图等高级可视化（v2）。

## 3. 三端适配策略（核心）

**一套响应式 SPA**，按断点切换布局，而非做三个独立 App。

| 设备 | 断点 | 导航形态 | 内容布局 | 终端 |
|---|---|---|---|---|
| 📱 手机 | `< 768px` | **底部 Tab Bar**（4 项）+ 详情页全屏覆盖 | 单列、卡片堆叠、大按钮 | 全屏页 + 底部快捷键栏 |
| 📲 平板 | `768–1199px` | **左侧可收起 Sidebar** | 两栏（列表 + 详情并排） | 右栏面板，可横屏全屏 |
| 💻 电脑 | `≥ 1200px` | **常驻左侧 Sidebar** | **三栏**（导航 + 列表 + 详情/终端），多面板 | 内嵌面板，可多终端分屏 |

> 断点对齐 Antd Grid：`xs <576 / sm ≥576 / md ≥768 / lg ≥992 / xl ≥1200 / xxl ≥1600`。
> 实践三档：mobile `< md`、tablet `md–xl`、desktop `≥ xl`。

设计准则：
- **移动优先 + 渐进增强**：先保证手机能用，宽屏逐步把"详情页"从覆盖层变为并排栏。
- **同一组件，不同容器**：列表、详情、终端都是独立组件，由布局容器（`AppShell`）按断点决定如何排布。
- **触控与鼠标双适配**：按钮 ≥ 44px 触控区；桌面提供键盘快捷键与右键菜单。
- **横竖屏感知**：手机横屏时终端自动进入沉浸模式。

```
 手机 (<768)         平板 (768-1199)              电脑 (≥1200)
┌──────────┐    ┌────┬───────────────┐    ┌────┬─────────┬──────────────┐
│  内容    │    │ ☰ │ 列表 │ 详情   │    │导航│  列表   │   详情/终端   │
│  (单列)  │    │导航│      │        │    │    │         │              │
│          │    │栏  │      │        │    │ ●  │ build✅ │  [xterm.js]  │
│          │    │    │      │        │    │ ●  │ test🏃 │              │
├──────────┤    │    │      │        │    │ ●  │ lint❌ │              │
│ Tab Bar  │    └────┴───────────────┘    └────┴─────────┴──────────────┘
└──────────┘
```

## 4. 技术选型

| 层 | 选型 | 理由 |
|---|---|---|
| 后端 | **Go + Gin**，编译为单静态二进制 `ttmux-web` | Gin 路由/中间件成熟；Go 跨平台、零依赖、可 `embed` 前端，一个文件丢服务器即跑 |
| 前端框架 | **React + Vite** | 生态成熟、Vite 构建快、HMR 体验好 |
| UI 组件 | **Ant Design (antd)** | 组件全、表格/表单/布局开箱即用；`Layout`+`Grid(Row/Col)` 响应式断点天然支持三端 |
| 移动适配 | antd 响应式 Grid + 自定义 `BottomTabBar`（或按需引入 `antd-mobile` 的 TabBar） | 手机底部导航 antd 本身没有，单独补 |
| 终端 | **xterm.js** + WebSocket（PTY 桥接） | 浏览器真实终端，支持 agent 交互/批准 |
| 实时 | **SSE**（状态推送）+ **WebSocket**（终端/日志流） | SSE 单向简单，WS 双向终端 |
| 鉴权 | 单口令 → 签名 HttpOnly Cookie（见 [03-auth-security](./03-auth-security.md)） | 简单安全 |
| 远程 | **Tailscale / Cloudflare Tunnel**（推荐）或反代 + HTTPS | 不开放公网端口 |

> **核心原则**：后端是 ttmux CLI 的**薄封装**，不重实现逻辑。
> 前端 Vite 构建产物由 Go `embed` 进二进制，单文件部署。

## 5. 系统架构

```
  ┌──────────────────────────┐      HTTPS / WSS      ┌────────────────────────────────┐
  │  浏览器 (手机/平板/电脑)  │ ◀── Tailscale/Tunnel ─▶ │       ttmux-web (Go + Gin)      │
  │  ┌────────────────────┐  │                       │  ┌──────────┐  ┌─────────────┐  │
  │  │  React + antd SPA  │  │   GET/POST  (JSON)    │  │Gin Router│  │ Auth 中间件 │  │
  │  │  AppShell(响应式)   │  │ ─────────────────────▶│  └────┬─────┘  └─────────────┘  │
  │  │  + xterm.js        │  │   SSE: 状态推送       │       │ exec / pty               │
  │  └────────────────────┘  │   WS : 终端/日志      │  ┌────▼──────────────────────┐  │
  └──────────────────────────┘                       │  │  ttmux CLI (bash)          │  │
                                                      │  │  tmux server               │  │
                                                      │  │  ~/.local/share/ttmux/     │  │
                                                      │  └────────────────────────────┘  │
                                                      └────────────────────────────────┘
```

- **读路径**：Gin handler → `ttmux <cmd> --json` → 解析 → JSON 返回。
- **写路径**：handler → 校验入参 → `exec.Command("ttmux", ...)` → 返回结果。
- **终端路径**：WS → 后端 `pty.Start("tmux","attach","-t",sess)` → 字节双向桥接到 xterm.js。
- **日志路径**：WS/SSE → tail `~/.local/share/ttmux/logs/<sess>.log` 流式推送。

详见 [02-backend](./02-backend.md) 与 [04-frontend](./04-frontend.md)。

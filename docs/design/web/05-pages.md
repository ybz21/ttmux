## P2. Sessions 列表 + 详情 `/sessions`
```
 手机(列表→全屏详情)        电脑(三栏并排)
┌──────────────────┐    ┌────┬─────────┬──────────────┐
│ Sessions         │    │导航│ Sessions│ build 详情    │
│ build   🏃       │    │    │ build🏃 │ status / cmd  │
│ deploy  ⏸️       │    │    │ deploy⏸│ [进终端]      │
│ test    ✅       │    │    │ test ✅ │ [capture]     │
│ [+ New]          │    │    │ [+ New] │ [send] [kill] │
└──────────────────┘    └────┴─────────┴──────────────┘
```
- 行：名称 + 状态 `Tag` + 当前进程 + 创建时间（手机用 `List`，电脑用 `Table`）。
- 详情：状态详情、capture 预览、操作按钮（进终端 / send / kill）。
- 手机点行进全屏详情（路由）；电脑右栏并排，选中高亮。

## P3. Tasks 列表 + 详情 `/tasks`（命令 + Agent 统一）
> ⭐ ttmux v0.4 起命令任务与 Agent 任务**统一为一个模型**（每个任务带 `type: cmd|agent`），
> Web 端对应**合并为一个 Tasks 页**，不再分 Groups / Agents 两个 Tab。
```
 手机                          电脑(三栏)
┌──────────────────┐    ┌────┬─────────┬──────────────────────┐
│ Tasks            │    │导航│ Tasks   │ refactor 组          │
│ build  🏃 2/3    │    │    │ build🏃 │ 任务   类型 状态 退出 │
│ refactor 🤖 1/3  │    │    │ refac🤖 │ api   🤖   🏃        │
│ ci     ✅ 3/3    │    │    │ ci   ✅ │ db    🤖   ✅  0      │
│ [+ 创建任务]     │    │    │         │ tests 🤖   ❌  1      │
└──────────────────┘    │    │         │ [日志][收集][进终端]  │
                        │    │         │ [追加指令] [清理]     │
                        └────┴─────────┴──────────────────────┘
```
- **列表**：组名 + 聚合状态 `Tag` + 进度 `done/total`（`Progress`）。组内若含 Agent 显示 🤖 标记。
- **详情**：任务 `Table`，**类型列**用图标区分 `⌨️ 命令` / `🤖 Agent`（对应 status `--json` 的 `type` 字段）；
  列含 名称 / 类型 / 状态 / 退出码 / 时长。
- **行内操作随类型自适应**：所有任务都有 日志 / 收集 / 进终端 / 追加指令(send) / 清理；
  Agent 行额外突出"进终端 / 追加指令"（批准场景）。
- **创建**：`+ 创建任务` 打开统一 Spawn 表单，先选类型（命令 / Agent），见 P9。
- 数据源：`GET /api/tasks`（=group ls）、`GET /api/tasks/:g`（=status --json，含 type）、
  `GET /api/tasks/:g/collect`。

## P5. ~~Agents~~ → 已并入 P3 Tasks
> Agent 不再是独立页面。Agent 任务就是 `type=agent` 的任务，与命令任务同列在 **Tasks** 页，
> 用 🤖 图标区分。cc-swarm 的监护/批准场景通过 Tasks 详情里 Agent 行的
> **「进终端 / 追加指令」**入口完成（电脑端右栏可内嵌终端预览，无需跳转即可批准危险操作）。
>
> 这与 CLI 的合并一致：`ttmux agent status/collect/kill/send` 现在都是
> `ttmux status/collect/group kill/send` 的别名。

## P6. 终端页 `/terminal/:name`
xterm.js 真实终端，批准 agent 的关键场景。
```
 手机(全屏+快捷键栏)         电脑(嵌入右栏/可多开 Tabs)
┌──────────────────┐    ┌────┬──────────┬─────────────┐
│ build  ● 已连接   │    │导航│ 列表     │ 终端1 │终端2 │
│ ┌──────────────┐ │    │    │          │[xterm]      │
│ │ [xterm.js]   │ │    │    │          │             │
│ │ Claude: 执行 │ │    │    │          │ Claude:...  │
│ │  rm -rf ..?  │ │    │    │          │             │
│ └──────────────┘ │    │    │          │ 物理键盘直接 │
│ [Yes][No][↵][⎋] │    │    │          │ 输入        │
│ [^C][↑][↓][Tab] │    └────┴──────────┴─────────────┘
└──────────────────┘
```
- **手机**：全屏 + 底部 `TermShortcutBar`（`Yes/No/Enter/Esc/Ctrl-C/↑/↓/Tab`）——
  移动端无物理键，这是刚需；横屏进沉浸模式。
- **电脑**：嵌入右栏，物理键盘直接输入；antd `Tabs` 支持多终端分屏/标签。
- resize 自动同步 PTY（FitAddon + ResizeObserver）；断线自动重连并提示。

## P9. Spawn 表单（手机 `Drawer` 底部弹出 / 电脑 `Modal`）
```
┌──────────────────────────────┐
│ Spawn          [命令]｜[Agent]│
│ 组名 ┌────────────────────┐   │
│      └────────────────────┘   │
│ 任务1 name ┌──┐ cmd ┌──────┐  │
│ 任务2 name ┌──┐ cmd ┌──────┐  │
│            [+ 加一行]         │
│ (Agent模式: dir/model/perm/  │
│  max-turns)                  │
│        [取消]   [Spawn]      │
└──────────────────────────────┘
```
- antd `Form.List` 动态增减任务行；模式切换 命令/Agent（`Segmented`）；Agent 模式展开额外字段；
  提交后跳对应详情页。

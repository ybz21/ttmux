# 蜂群成员角色模型设计（细分角色 + 职责）

> 前置：建立在 [蜂群编排设计.md](蜂群编排设计.md)（swarm/member/leader/依赖门控）、
> [蜂群成员启动与提示词模板设计.md](蜂群成员启动与提示词模板设计.md)（kind/role 列 + text/template 渲染）
> 与 [蜂群 Web 接入设计.md](蜂群%20Web%20接入设计.md) 之上。本设计**扩展成员身份模型**，不改既有协作底座。
>
> 状态：v1.0 —— **已实现**（§9 四阶段全部落地）。一处与设计的细微差异：加成员弹窗的
> 细分角色是「枚举下拉」（暂未做 UI 自定义输入），自定义 subrole 仍可经 CLI `--subrole`
> 写入并在界面正常展示。
>
> 已定决策（三选一已拍板）：
> - 细分角色取值 = **固定枚举 + 自定义兜底**
> - prompt 组织 = **单模板 + 角色片段**（`worker.md.tmpl` 参数化 + `prompts/roles/<key>.md`）
> - 职责与任务 = **两者都保留**（duty=长期职责，task=本次起步任务）

## 1. 背景与目标

当前一个蜂群成员只有粗角色 `leader|member`，"它是干嘛的、负责哪块"全被塞进自由文本
`task` 里。结果：拓扑图上一排无差别的 `member`，leader 拆活时也没有"班子"概念，prompt
对所有 worker 一视同仁。

本设计给每个成员补两个一等公民属性：

1. **细分角色 `subrole`**：产品经理 / 架构师 / 前端 / 后端 / 测试 / 设计 / 审查 / 运维 …（固定枚举 + 自定义）。
2. **职责 `duty`**：这个成员长期负责哪一块、产出什么标准（区别于"本次具体起步任务" `task`）。

并把这两者贯通**底层存储 → CLI/API → 前端展现 → 提示词模板 → leader 拆活**全链路。

彼此独立；`type=task`（纯 shell）时 subrole/duty 不适用。

## 3. 细分角色注册表

源头放后端 `backend/api`（规范 key + 角色片段文件名同 key）；前端用同一 key 配图标 + i18n 文案；
CLI 只做轻量归一。**未命中 key = 自定义**，原样存、走通用处理。

| key | 中文 | 图标 | 侧重 |
|-----|------|:--:|------|
| `pm` | 产品经理 | 🧭 | 需求 / 范围 / 验收标准 |
| `architect` | 架构师 | 🏛 | 技术方案 / 接口契约 / 分层 |
| `frontend` | 前端工程师 | 🎨 | UI 实现 |
| `backend` | 后端工程师 | ⚙️ | 服务 / API / 数据 |
| `fullstack` | 全栈工程师 | 🛠 | 端到端 |
| `qa` | 测试工程师 | 🧪 | 用例 / 回归 / 验收 |
| `designer` | 设计师 | ✏️ | 视觉 / 交互 |
| `reviewer` | 代码审查 | 🔍 | review / challenge |
| `devops` | 运维 | 🚢 | 构建 / 部署 / CI |
| `docs` | 文档 | 📝 | 说明 / 文档 |
| `commander` | 总指挥 | ◆ | **leader 专属默认** |
| *(其它)* | 自定义 | 👤 | 原样存，通用 prompt |

`SubroleNorm(s)`：把中英文别名（"工程师"/"engineer"/"eng" → `backend`/`fullstack` 等）归一到 key；
未识别保留原串。CLI 与后端各持一份最小别名表；前端的图标/文案表以 key 对齐。

> **单一源头**：后端暴露 `GET /api/swarm/subroles → [{key,label,icon}]` 供前端下拉与展示，
> i18n label 仍落在前端 locale；角色 prompt 片段是 `prompts/roles/<key>.md` 文件。三处都以 key 锚定，避免漂移。

## 4. 底层实现

### 4.1 schema 迁移

沿用 kind/role 的 `tableColumns` 守卫套路（`swarm.go`），旧库平滑加列：

```sql
ALTER TABLE members ADD COLUMN subrole TEXT DEFAULT '';  -- 细分角色 key（空=未指定/自定义）
ALTER TABLE members ADD COLUMN duty    TEXT DEFAULT '';  -- 长期职责（自由文本）
```

`initSwarmDB`（`store.go:154`）的 `CREATE TABLE` 同步加这两列。空值 → 全链路有兜底，老蜂群不受影响。

### 4.2 Go 结构体 / 读写

- `MemberSpec`（`write.go:11`）+= `Subrole`、`Duty`；`AddMemberRow`/`SetPending`/`pendingSpec` 的
  INSERT/SELECT 带上两列。
- `SwarmMember`（`swarm.go:38`，JSON）+= `Subrole`、`Duty`；status 查询 SELECT 带上（`swarm.go:231`）。

### 4.3 CLI

`swarm add`（`command/swarm/swarm.go:158`）新增旗标：

```bash
ttmux swarm add 航母1 ui --type agent \
  --subrole frontend --duty "负责态势看板的前端实现与交互" \
  "搭三栏外壳 + 接 window.Sim 契约渲染"
```

解析 `--subrole`/`--duty`，`SubroleNorm` 归一后写库。`--subrole` 缺省：leader 行 → `commander`，
member 行 → 空（通用）。

### 4.4 后端 API

- `SwarmAddMember`（`swarm.go:57`）：body += `subrole`/`duty`，透传给 `ttmux swarm add`，并喂给 prompt 渲染。
- 新增 `GET /api/swarm/subroles`：返回注册表（key/label/icon），前端下拉用。

## 5. Prompt 组织（单模板 + 角色片段）

```
backend/api/prompts/
  worker.md.tmpl          # 基底：参数化 {{.SubroleLabel}} {{.Duty}} {{.RoleTrait}}
  master.md.tmpl          # 已有
  auto_leader.md.tmpl     # 已有
  roles/                  # ← 新增：每个细分角色一段"工作方式 / 产出标准"
    pm.md  architect.md  frontend.md  backend.md  fullstack.md
    qa.md  designer.md  reviewer.md  devops.md  docs.md
```

**渲染流程**（`prompt.go`）：`renderMemberPrompt` 先 `SubroleNorm(ctx.Subrole)` →
读 `prompts/roles/<key>.md`（命中才有；自定义/未命中 → 空）→ 内容塞入 `ctx.RoleTrait` →
渲染 `worker.md.tmpl`。`promptCtx` += `Subrole`(key)、`SubroleLabel`(中文)、`Duty`、`RoleTrait`。

**`worker.md.tmpl` 头部改造**：

```markdown
# {{.Member}} · {{with .SubroleLabel}}{{.}}{{else}}成员{{end}}（{{.Role}}）

**蜂群**：{{.Swarm}}{{if .Goal}} ｜ **目标**：{{.Goal}}{{end}}
{{if .Duty}}**职责**：{{.Duty}}{{end}}

## 你的起步任务
{{.Task}}
{{if .RoleTrait}}
## 你的工作方式（{{.SubroleLabel}}）
{{.RoleTrait}}
{{end}}
（以下为已有的：协作方式 / 广场 / 看板 / done 段，保持不变）
```

**角色片段示例**（`roles/qa.md`）：

```markdown
- 先读被测产物的验收标准，再动手；测试覆盖正常 / 边界 / 异常三类。
- Web 类成果用 `chrome` 真跑断言（开页面、点按钮、读文本），别只读代码。
- 发现缺陷用 `swarm say --kind block` 回报给负责人，**不替对方改实现**。
```

> 自定义/未填 subrole 的成员：`SubroleLabel` 回退为"成员"、`RoleTrait` 为空 →
> 退化成今天的通用 worker prompt，行为不破。

### 6.2 办公室 desk —— 细分角色"穿在身上"

把细分角色编码进**小人形象**，职责放**悬浮**，一眼看清"班子构成"：

1. **帽子 = 细分角色**（核心）。把 `memberHatColor`/`memberHatStyle`（`Swarm.tsx:603`）从
   "按 name 随机" 改为 **按 subrole 取**：
   - 颜色 = 注册表（§3）的 subrole 色；leader 仍红 + `hat-leader`(王冠)。
   - 帽形 = 新增 `hat-cap`(鸭舌·工程/前端) / `hat-round`(圆顶·产品/审查) / `hat-flat`(平顶·测试/运维) 等
     变体（CSS 在 `index.css` 办公室块内补，对照 mockup）；自定义/未知 → 回退现有 `stableIndex` 帽。
2. **桌前名牌 = 细分角色**。在 `swarm-office-label` 里名字与状态之间插一枚 `.swarm-office-role`
   小药丸（图标+文案，色同帽 `--role-color`）：`[◆ cc-航母1] [🎨 前端] [running]`。
3. **职责 = 悬浮**。desk 容器 `title={duty}`（hover 显示）；完整 `duty`+`task` 进抽屉。
4. 显示器(引擎 claude/codex)、状态药丸/动画、统计——**全部不变**。

```
        ◆ cc-航母1  [◆总指挥]            pm [🧭产品]   arch [🏛架构]   api [⚙️后端]   ui [🎨前端]   qa [🧪测试]

          └──🧑──┘                       帽=青         帽=紫          帽=蓝         帽=粉        帽=黄
       (帽色/帽形 = 细分角色；hover desk → 职责 tooltip；点击 → 右侧抽屉)

### 6.3 点击成员 → NodeDrawer（在现有抽屉上加两段）

沿用 `NodeDrawer`（`Swarm.tsx` 的 antd 右侧 `Drawer`），增量：

- **标题**：名字 + 状态 Tag 之外，再加一枚**细分角色徽章**（`🎨 前端工程师`）。
- **身份段**（已有）：补"细分角色 = …"，其余(role/type/引擎/终端/依赖)不变。
- **新增「职责(DUTY · 常驻)」段**：高亮框显示 `duty`，放在身份之后、初始指令之前。
- **「初始指令 / 当前任务」段**（已有 `initCmd`/`m.task`）：标题点明这是渲染自 `worker.md.tmpl`
  的 prompt（含职责/工作方式），与上面的"职责"区分开。
- 看板卡 / 广场 / 操作（进终端·done·解锁）——**不变**；操作区可加「改细分角色/职责」入口（可选，后置）。

```
┌ 抽屉：ui  [🎨 前端工程师]  running ───────────────┐
│ 身份  agent·member·细分角色 前端工程师·引擎 claude·终端 航母1-ui │
│       [进终端↗] [改细分角色/职责] [标完成 done]    │
│ 职责(DUTY·常驻)  ▏负责态势看板的前端实现与交互     │  ← 新增高亮段
│ 初始指令/当前任务  # ui · 前端工程师(member) …(prompt 全文) │
│ 看板卡(2) · 广场(6)                               │
└───────────────────────────────────────────────────┘
```

### 6.4 加成员弹窗

在现有「类型 / 引擎 / 名字 / 任务 / 目录 / 依赖」基础上新增两项：

```

## 7. Leader 集成（拆活自动带身份）

让 leader 拆任务时就给每个成员定**细分角色 + 职责**，拆出来的是"班子"而非一堆无差别 member。

- `auto_leader.md.tmpl` / cc-swarm `docs/decompose.md`：派活示例改成带 `--subrole`/`--duty`：

  ```bash
  ttmux swarm add <群> <名> --type agent --subrole <key> --duty "<负责什么>" "<起步任务>"
  ```
- 在拆任务一步明确要求："为每个子任务指定细分角色（产品/架构/前端/后端/测试…）与职责"。

## 8. 兼容与迁移

- 旧蜂群：`subrole`/`duty` 空 → UI 显示大角色、prompt 走通用 worker，无破坏。
